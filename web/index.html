LLM social deduction · hidden-role Werewolf

Which model is the best liar — and which is best at catching liars?

Six free language models are seated at a table with hidden roles. Two are werewolves who know each other and kill by night; one seer quietly learns alignments; one doctor shields a seat; two villagers just reason and vote. By day they argue in public and vote one player out. You watch the day-chat live, then flip the director's cut to read what each model was really thinking.

Two rules hold the whole thing up

§3.1 · firewall

Information firewalling

An agent only ever sees its role-scoped view, viewFor(game, seat). The wolf prompt knows the co-wolf and the night kill; the villager prompt must not. Hidden state never leaks through a shared history object — it's the per-user authorization-of-context problem, done properly.

§3.2 · authority

The engine is truth; the model advises

The model proposes a move; a deterministic TypeScript engine validates and applies it. You can't vote a dead seat or kill as a villager. Illegal or unparseable output falls back to a legal move and is recorded as a violation — never enforced by the model.

Build status

Greenfield, built in the order the design doc insists on: a correct, fully-tested engine before a single model call.

  • M0 — console spike, dummy agents, 6 players (npm run spike)
  • M1 — deterministic engine + viewFor firewall + 14 passing unit tests
  • M2 — real models via OpenRouter (+ parse / repair / retry)
  • M3 — persistence (Neon) + post-game stats
  • M4 — live SSE UI on Deno Deploy
  • M5 — leaderboard + ELO + director's-cut view
  • M6 — polish: personas, shareable replays
